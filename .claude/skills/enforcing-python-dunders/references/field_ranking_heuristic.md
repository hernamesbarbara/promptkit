# Field Ranking Heuristic for __repr__ and __str__

This reference provides general-purpose guidelines for selecting high-value fields to include in Python `__repr__` and `__str__` methods. These rules help the skill highlight the most meaningful identifying fields in business-oriented models while staying domain-neutral.

---

## 1. Priority Ranking of Fields

Rank fields by their usefulness for recognizing the object at a glance.

### **1. Primary Identifiers**
Uniquely identify the entity.

Common names:
- `id`, `<entity>_id`
- `uuid`, `guid`
- `slug`, `code`
- `ticker`, `iso_code`

### **2. Human-Readable Names**
Provide a natural, recognizable label.
- `name`, `full_name`, `company_name`
- `title`, `display_name`, `label`, `headline`

### **3. Canonical External Handles / Links**
Connect the entity to external systems.

Examples:
- `*_url`, `*_link`, `canonical_*`
- `website`, `domain`
- `email`, `username`, `handle`

### **4. Compact Context / Status Indicators**
Short descriptors that convey immediate meaning.
- `status`, `state`, `stage`
- `type`, `category`, `kind`
- `size_range`, `level`, `role`
- `country`, `region`, `location`

### **5. One Optional Date**
Include only if central to identity or chronology.
- `created_at`
- `updated_at`
- `scheduled_for`

---

## 2. Fields to Avoid in Representations

### **1. Large Text Fields**
Avoid long text fields, such as:
- `description`
- `summary`
- `metadata_*`
- comment/notes fields

### **2. Large Collections**
Summarize instead of printing contents.
Examples:
- `technologies=[... 17 items ...]`
- `locations_full=<3 locations>`

### **3. Internal or System Metadata**
Exclude fields meant for internal use:
- `raw_*`, `internal_*`, `_debug_*`
- `enriched_*`, autogenerated metadata

---

## 3. Heuristics for __repr__

`__repr__` should:
- show **2-5 high-value fields**
- always include an identifier and a human readable name
- optionally include:
  - a canonical link, or
  - a compact status/type field

Example pattern:
```
ClassName(id=..., name=..., handle=..., status=...)
```

Avoid dumping every field in large models.

---

## 4. Heuristics for __str__

`__str__` should:
- be concise and human-friendly
- start with the primary display name
- include 1-2 short qualifiers (industry, size, type, region)
- avoid raw IDs unless essential

Example pattern:
```
"Acme Inc (SaaS, 51-200 employees) - United States"
```

---

## 5. When Uncertain

If unclear which fields matter most:
- default to: `id` + `name`
- optionally add a single contextual field (`status`, `type`, `region`)
- or include a TODO comment for human review
